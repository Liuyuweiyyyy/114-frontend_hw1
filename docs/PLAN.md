# GitHub 偵察機 - 製作計畫

大家好！今天我們要一起做一個很有趣的網頁，叫做「GitHub 偵察機」！

這個網頁可以做什麼呢？
就是當我們輸入一個 GitHub 叔叔或阿姨的名字，它可以幫我們找到那個人的照片和資料，就像偵探一樣厲害！

## 照片是怎麼出現的？(中間經過的關卡)

想像一下便利商店的服務台：

### 關卡1：寫下名字
[           輸入 GitHub 名字           ]
就像在表格上寫下要找人 的名字

### 關卡2：按下按鈕
[ 按下搜尋 ]
就像把表格交給服務人員：「我要找這個人！」

### 關卡3：觸發事件
服務人員收到表格，開始工作
（在程式碼中叫 handleSearch 被叫起來）

### 關卡4：顯示「等一下」
螢幕上出現「正在搜尋中...」
讓你知道服務人員正在打電話

### 關卡5：打第一通電話
服務人員打去 GitHub 的圖書館：
「可以把這個叔叔的資料給我嗎？」
（fetch 去 api.github.com/users/名字）

### 關卡6：收到回覆
GitHub 說：「找到了！這是他的照片、名字、統計資料」
（拿到 JSON 資料）

### 關卡7：打第二通電話
服務人員再打去問：
「這個叔叔有什麼公開的專案？」（fetch 去 api.github.com/users/名字/repos）

### 關卡8：收到回覆
GitHub 說：「他有三個專案，这是專案清單！」（拿到專案資料）

### 關卡9：資料放進盒子
服務人員把收到的資料，放進盒子裡
（在程式碼中：searchResult.value = data）

### 關卡10：畫面更新
Vue 看到盒子裡有資料了，就更新畫面
（v-if="searchResult" 變成顯示）

### 關卡11：照片出現！
畫面上就出現照片啦！
（<img :src="searchResult.avatar_url">）

---

## 第一步：做搜尋的框框

要先做一個可以打字的地方，就像這樣：
[           輸入 GitHub 名字           ] [ 按下搜尋 ]

這樣我們才能輸入要尋找的人的名字。

## 第二步：連線去 GitHub 抓資料

就像打電話一樣，我們要打去 GitHub 的圖書館（API）
問他們：「可以把這個叔叔的資料給我嗎？」

GitHub 會告訴我們：
- 叔叔的長相（頭像）
- 叔叔叫什麼名字
- 叔叔住在哪裡
- 有多少人追蹤叔叔

## 第三步：顯示公開專案

打好資料電話之後，我們還要打另一通電話去問：
「這個叔叔有哪些公開的專案呢？」

GitHub 會告訴我們：
- 專案叫什麼名字
- 有多少人按讚（星星數）
- 專案是做什麼的（描述）

然後把這些專案變成一個清單，讓大家可以看！

## 第四步：把資料變成卡片

把拿到的資料變成一張美美的卡片，上面有：
- 頭像（照片）
- 名字
- 簡介
- 公開專案數
- 追蹤者數字
- 追蹤中數字

## 第五步：處理錯誤

如果找不到這個人怎麼辦？
我們要顯示一個提示，告訴大家「找不到這個小朋友」！
